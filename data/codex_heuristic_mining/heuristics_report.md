- Severity: висока
- Support: 8 tenders
- Confidence: high
- Detection pattern: (?is)(еквівалент|аналог).{0,250}(ідентичн|повністю\s+відповіда|співпада|збігає|без\s+відхилень|100\s*%\s*відповідн|будь-як\w*\s+відмінност\w*.{0,80}відхиля)
- False-positive guards: Не спрацьовувати, якщо йдеться про відповідність мінімальним вимогам, прямо задані допустимі діапазони/відхилення або критерії функціональної еквівалентності.
- Why: Фраза «або еквівалент» формально є, але вимога повної тотожності може нівелювати конкуренцію.
- Rewrite: Визначити мінімальні суттєві характеристики і допустимі відхилення, які не погіршують функціональність товару.
- Implementation: Підвищувати severity, якщо поруч є бренд/модель або 5+ точних параметрів у технічній таблиці.
- Evidence phrases:
  - еквівалент не повинен відрізнятися
  - ідентичними або більш якісними характеристиками
  - співпадає по всім технічним характеристикам
  - всі характеристики товару повинні співпадати

- Detection pattern: (?is)(прийнятн\w*\s+країн|Ukraine\s+Facility|сертифікат\w*\s+походження|деклараці\w*\s+походження|довідк\w*\s+про\s+походження|країна\s+походження).{0,160}(кожн\w*\s+(номенклатурн\w*\s+)?позиці|кожн\w*\s+товар|від\s+виробника)
- False-positive guards: Не спрацьовувати на нейтральне поле `країна походження` або санкційну декларацію без вимоги документа/позиційності; знижувати severity, якщо вимога лише на етапі поставки і є чіткі альтернативи.
- Why: Позиційне підтвердження походження може бути значним документальним бар’єром.
- Rewrite: Визначити перелік прийнятних документів, етап подання, винятки та еквівалентні способи підтвердження.
- Implementation: Для Ukraine Facility лишати як review-needed, бо може бути обґрунтовано фінансуванням.
- Evidence phrases:
  - усі товари ... походять із прийнятних країн
  - документ ... щодо кожної номенклатурної позиції
  - довідка про походження від виробника за кожною позицією
  - країна походження товару щодо кожної номенклатурної позиції

### 29. секторні фінансові вимоги: Страхові рейтинги, топ-місця або регуляторні довідки як жорсткі пороги участі

- Severity: середня
- Support: 5 tenders
- Confidence: high
- Detection pattern: For insurance CPV/terms: (?is)(МТСБУ|страховик|страхування|НБУ).{0,180}(топ[-\s]?5|перша\s+п.?ятірка|Q\s*=?\s*5|рейтинг\w*.{0,60}uaAА|uaAA|частка\s+вимог.{0,80}не\s+менше\s+\d+\s*%|довідк\w*\s+НБУ.{0,120}заход\w*\s+вплив)
- False-positive guards: Не спрацьовувати на довідкову статистику без обов’язку для учасника, або якщо вилучено змінами; у такому разі маркувати як усунений ризик.
- Why: Жорсткі рейтингові/регуляторні пороги можуть звужувати коло ліцензованих страховиків.
- Rewrite: Замінити на обґрунтовані мінімальні показники спроможності або дозволити еквівалентні підтвердження.
- Implementation: Секторне правило, але достатньо повторюване й детерміноване для страхових закупівель.
- Evidence phrases:
  - перша п’ятірка Страховиків
  - Q=5
  - рейтинг не нижче uaAA
  - довідку НБУ про відсутність заходів впливу
  - частка вимог ... не менше 80%

### 30. узгодженість документів: Різні строки, місця або залишкові строки придатності в різних документах одного тендера

- Severity: середня
- Support: 5 tenders
- Confidence: high
- Detection pattern: Extract normalized values by role: delivery deadlines, performance place топоніми, shelf-life percentages. Flag if same role has 2+ distinct values across docs without hierarchy markers (`партія`, `етап`, `кінцевий`, `проміжний`, `за заявкою`).
- False-positive guards: Не порівнювати з датами аукціону, строком дії пропозиції, договору, оскарження чи різними групами товарів, якщо це явно структуровано.
- Why: Суперечності створюють неоднакове розуміння умов участі та виконання.
- Rewrite: Уніфікувати умови в оголошенні, специфікації та договорі або прямо визначити пріоритет і сферу кожного строку.
- Implementation: Потребує tender-level extraction, не локального regex; високий precision для дат DD.MM.YYYY і відсотків біля `придатн`.
- Evidence phrases:
  - до 31.08.2026 року / протягом 3 календарних днів
  - до 15.11.2026 / до 31.12.2026
  - 70% / 80% терміну придатності
  - Стрийська / Сокальська громада

## Existing Rule Refinements

- бренд/модель без «або еквівалент»: Додати підтипи: пряма заборона еквівалентів; бренд/модель із слабким обґрунтуванням `оптимально відповідає потребам`; брендові компоненти всередині товару (`компресор Secop`, `термостат Dixell`); конкретні ОС/антивіруси/ПЗ як частина комплектації.
  Reason: Ці випадки можуть не виглядати як простий рядок `бренд без еквівалента`, але мають той самий або сильніший обмежувальний ефект.
- лист виробника: Розширити на вимоги копії договору/правовідносин із виробником, офіційним представником, імпортером або листа про співпрацю, особливо `до підписання договору` чи `якщо постачальник не виробник`.
  Reason: Це функціонально той самий ланцюговий бар’єр, навіть якщо слово `лист виробника` не використане.
- географічне обмеження: Додати непрямі логістичні аналоги: обов’язковий виїзд/прибуття представника за 2-3 дні або 2 години, фізична присутність при прийманні, заборона доставки перевізником.
  Reason: Такі умови можуть мати ефект локальної прив’язки без прямої вимоги офісу/складу в регіоні.
- документальні вимоги: Додати маркери посторінкового підпису/печатки, мокрої печатки без `за наявності`, `сканкопія оригіналу`, нотаріальний переклад/апостиль для нерезидентів, свіжість документів `не раніше 30 днів` як низько/середньо пріоритетні підтипи.
  Reason: Це формалізовані документальні бар’єри, близькі до вже наявних правил про оригінали/нотаріальні копії.
- умови оплати/поставки: Відокремити `нечітко/відсутньо` від `чітко, але обтяжливо`: післяоплата 60+ днів, бюджетна відстрочка без межі, оплата після реєстрації ПДВ, невідомі адреси до заявки/переможця.
  Reason: Наявне правило ловить неясність, але не ловить пропорційні ризики за чітко сформульованих умов.
- кваліфікаційні вимоги: Додати не лише високі роки/кількість договорів, а й змістове звуження: бюджетні кошти, конкретний тип об’єкта, фіксований вузький період, обов’язкові позитивні відгуки.
  Reason: Ці обмеження можуть бути сильними навіть без порогу `3+ роки/договори`.
- нечітка вимога: Додати підтип `дискреція замовника`: `еквівалентність визначається Замовником`, штраф `% визначається на розсуд Замовника`, `інші заходи впливу на власний розсуд`.
  Reason: Це не просто нечіткість, а формалізована дискреція в оцінці або санкціях.

## Rejected Or Low-Signal Patterns

- Загальна вимога нового/невживаного товару без конкретного року виробництва або без зв’язку з гарантією.
- Нейтральне поле `країна походження` без обмеження допустимих країн або документів для кожної позиції.
- Стандартна вимога зареєструвати податкову накладну без права замовника затримати оплату чи утримати ПДВ.
- Звичайне право розірвання договору за згодою сторін, рішенням суду або істотним порушенням із строком на усунення.
- Посилання на форс-мажор без прямого виключення війни, електропостачання або обставин контрагентів.
- Наявність банківської гарантії сама по собі без розміру, подвійного забезпечення або обмеження кола банків.
- Сервісні центри без географічної прив’язки, якщо це лише контактна інформація і не вимагається власна/офіційна мережа.
- Витрати учасника на підготовку пропозиції, якщо формулювання не охоплює сторонні послуги замовника, поставку чи дискваліфікацію.
- Маркування Ukraine Facility / емблема ЄС без права відмови від приймання або нечітких фізичних вимог до товару.
- Посилання на П-6/П-7 як низькопріоритетний договірний ризик: детерміновано, але слабкий загальний сигнал.
- Збита нумерація списків у документації: корисно для QA документа, але слабко як антиконкурентна евристика.
- Антикорупційні анкети про зв’язки та судові спори: залишити для privacy/compliance review, не для основного набору закупівельних ризиків.
- Необмежена кількість змін ціни для пального: може бути нормативно допустимою і секторно специфічною; краще не вводити без юридичного контексту.
- Фіксований залишковий строк придатності 1 рік або дата придатності: сигнал слабкий без товарної моделі строків придатності; краще ловити лише суперечності між документами або явно нереалістичні пороги.
- Вимога санітарно-епідеміологічного висновку/протоколу для окремого будматеріалу: занадто предметно-специфічно для загального правила, якщо немає масового повторення або заборони альтернатив.
